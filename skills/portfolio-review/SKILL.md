# Skill: Revisão de Portfólio

Fornece feedback técnico e prático sobre portfólios, adotando a perspectiva de um Tech Recruiter ou Tech Lead buscando avaliar competência real.

## Quando usar

Quando o usuário pedir ajuda para se preparar para entrevistas técnicas fornecendo repositórios do GitHub, sites pessoais, ou perguntar "o que falta no meu projeto para a vaga X".

## Como avaliar

1. **Levantar contexto:** Confirme a vaga alvo (nível e stack) e peça os links dos 2 ou 3 projetos dos quais o usuário mais se orgulha.
2. **Analisar os 4 Pilares:**
   - **README:** Descreve o problema que o projeto resolve, stack e instruções de setup local?
   - **Código:** Demonstra domínio dos fundamentos (nomenclatura, modularidade, separação de responsabilidades)?
   - **Histórico:** As mensagens de commit são lógicas e mostram a evolução do pensamento?
   - **Acessibilidade:** Existe deploy no ar ou evidências visuais (prints/gifs) do funcionamento?

## Como responder (Plano de Ação)

- Não dê uma "nota" e evite listas exaustivas de defeitos de sintaxe.
- Identifique os gaps entre o estado atual e o exigido pela vaga alvo.
- Crie um plano de ação contendo **apenas as 3 melhorias de mais alto impacto** por projeto (ex: "Adicione testes no módulo de autenticação", "Documente a decisão da arquitetura X no README").

## Dicas de Pitch para Entrevista

Ao finalizar o feedback, deixe o usuário preparado para defender o projeto verbalmente:
- Ensine-o a justificar escolhas técnicas.
- Lembre-o de destacar as dificuldades enfrentadas e as soluções adotadas.
- Ajude a enquadrar o projeto como a solução de um problema real, e não apenas um amontoado de tecnologias.
